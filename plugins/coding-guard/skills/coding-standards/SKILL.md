# 编码规范速查

本技能由 `coding-guard` 插件提供，汇总通用编码安全规范。

## 文件编码

- 所有源码文件使用 **UTF-8 无 BOM** 编码
- 禁止使用 UTF-8 BOM、UTF-16、Unicode 或其他编码
- 新建文件统一使用 UTF-8 无 BOM

## 安全编码

- 禁止在代码中硬编码密码、token、API key、连接字符串
- 敏感信息使用环境变量或公共常量代替
- 提交前确认不包含 `password=`, `secret=`, `api_key=` 等明文

## 代码风格

- 保持与项目现有代码风格一致（对齐、命名、异常处理）
- 在已有模式下扩展，不随意引入新风格或依赖

## 修改原则

- 使用 Edit/Write 工具进行代码修改，禁止使用 `Set-Content`、`Out-File`、`>` 重定向等可能改变编码的命令
- 编辑前确认目标文件编码，编辑后验证未引入 BOM 头
- 做最小化修改，不顺手整理无关代码

## 环境适配

- 读取项目配置文件（`composer.json` / `package.json` 等）推断技术栈版本
- 生成代码必须兼容当前项目框架版本
